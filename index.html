01
What It Does

Three Modes of Operation

👀

PR Watcher

Monitors every channel it's invited to. When someone posts a GitHub PR link, it automatically reviews the code, cross-checks with Jira tickets, and gives an approve/request-changes verdict.

💬

DM Assistant

Personal AI assistant via direct messages. Routes your intent — ideas become Jira tickets, tasks become tracked work items, questions get answered with full context.

🏷️

@Mention Handler

Tag the bot in any thread to create a Jira ticket from the conversation context, update existing tickets, or even trigger code implementation.

04
Deep Dive

PR Review — Smart Features

🔄

Re-Reviews

Reply in the PR thread asking for a re-review. The bot detects intent, cleans up old reactions, and runs a fresh review with full thread context.

❓

Thread Q&A

Ask questions about the PR in the review thread. The bot uses thread history + PR diff to answer contextually — no need to repeat yourself.

🔀

Merged Detection

Already merged? The bot adds a 🔀 merge reaction and skips the review. No wasted compute or noisy threads.

🤖

Bot Message Support

Works with bot-posted PR links too (e.g., CI/CD bots). Detects its own thread replies to prevent infinite review loops.

Completion Guarantee

Every review requires a ---REVIEW_COMPLETE--- marker. If missing, the bot retries automatically. Failed reviews get a ⚠️ with "please review manually" fallback.

Channel Power

@Mention — Thread to Ticket Pipeline

Turn any Slack conversation into a tracked Jira ticket with one @mention.

💬 Thread Discussion
→
🏷️ @bot create ticket
→
📎 Files Attached
→
🔗 Slack Link in Ticket

What It Captures

  • Full thread conversation as ticket description
  • Attached files (images, docs) → Jira attachments
  • Permalink back to original Slack thread
  • Proper issue type (Story, Bug, Task, Spike)

Follow-Up Actions

  • @mention again → update the existing ticket
  • "implement it" → bot reads ticket, writes code, opens PR
  • "fix it" → same implementation flow
  • All actions tracked via thread context

07
Real World

Typical Use Cases

🏃 Sprint Planning

Team discusses a feature in Slack. Someone @mentions the bot: "create a spike for this". Bot reads the entire thread, creates a properly described Jira Spike with context and attachments.

🐛 Bug Triage

QA posts a bug with screenshots in a channel. @mention bot: "log a bug for this". Bot creates the ticket with screenshots auto-attached, severity inferred from discussion.

📝 Quick PR Feedback

Engineer posts a PR at 11pm. By morning, the bot has already reviewed the diff, cross-checked with the Jira ticket, and left detailed feedback in the thread — with approve/changes decision.

💭 Idea Capture

Random thought in DMs: "We should cache API responses client-side". Bot creates a Jira Story tagged (IDEA) so it doesn't get lost in the Slack void.

🔧 Quick Implementation

After ticket is created, reply "@bot implement it". Bot reads the ticket, finds relevant code, makes the fix, runs tests, and opens a PR — all from a Slack thread.

❓ PR Deep Dive

"Why did you flag this function?" — ask follow-up questions in the review thread. The bot re-reads the PR diff with your question context and gives a targeted explanation.

08
Visual Feedback

Reaction Language

The bot communicates PR status through emoji reactions — scannable at a glance.

👀
Eyes — Processing. Bot is actively reviewing the PR.
👍
Thumbs Up — Approved. Code looks good, recommend merging.
⚠️
Warning — Changes Requested. Issues found that should be addressed before merge.
🔀
Merge — Already Merged. PR was merged before review started; no action needed.
Re-Review Cleanup

When a re-review is triggered, the bot removes all previous reactions first, then adds fresh ones based on the new review. No stale signals.

09
Get Started

Setup in 3 Steps

1️⃣

Configure

Create .env with Slack tokens, GitHub token, Jira credentials. Or run ./claude and ask "help me bootstrap".

2️⃣

Launch

./claude slack — starts the Docker container with Slack Bridge in Socket Mode. No webhooks, no public URLs needed.

3️⃣

Invite

Add the bot to your channels with /invite @BotName. It starts watching for PR links and @mentions immediately.
